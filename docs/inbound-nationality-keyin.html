인바운드 거래처 국적 매핑 관리

57-prefix 회원번호(구코드)는 회원명에 국적이 표기되지 않아 자동 매핑이 어렵습니다. 이 페이지에서 거래처별 국적을 수동 매핑하면 build_inbound_enriched.py 빌드 시 우선 적용됩니다.
우선순위: 매핑불가 (UNMAPPABLE) > 회원명 (국적) 표기 > 키인 매핑 > 87 master > 이름 토큰 추정 > 미확인

🛠️ 사용법
  1. 아래 표에서 미확인 거래처의 국적을 드롭다운으로 선택 (또는 다국적 선택)
  2. 다국적 인바운드 거래처처럼 단일 국적 매핑이 부적합한 경우 → 🚫 매핑 불가 선택 후 사유 메모 입력
  3. 「JSON 내보내기」 클릭 → inbound_partner_nationality_keyin.json 다운로드
  4. 다운로드한 파일을 docs/data/ 폴더에 덮어쓰기 → 커밋 & 푸시
  5. 다음 빌드(scripts/build_inbound_enriched.py)에서 자동 반영
    • 매핑된 거래처 → 해당 국적 (키인매핑)
    • 매핑 불가 등재 거래처 → '매핑불가' 분류 (미확인과 분리 집계)
