업무 가이드
GUIDES

전체 흐름

1. 신규 등록 품의 페이지에서 기획전 등록 pumui.html
2. 상세 입력 대상 사업장·기간·요금·총 RN/REV 목표 입력 엑셀 업로드 가능
▼
3. KPI 자동배분 총 RN/REV/ADR을 일자별·사업장별로 자동 분할 공휴일·주말 가중치 반영
4. 구글시트 동기화 제출 시 Apps Script로 시트 적재 중복 방지 로직 포함
▼
5. Action Plan 반영 action_plan_dashboard.html에서 자동 로드 → 일자별 KPI 추적 실적 vs 목표 비교

등록 절차

1품의 페이지 접속 — pumui.html에서 "기획전 등록" 유형 선택
2기본 정보 입력 — 기획전명, 대상 사업장(다중 선택), 판매 기간, 적용 채널
3요금 및 KPI 총계 입력 — 패키지 코드, 요금안(min~max), 총 목표 RN/REV/ADR 입력
4엑셀 업로드(선택) — 다수 기획전 일괄 등록 시 템플릿 활용
5구글시트 제출 — "제출" 버튼으로 Apps Script가 Key시트에 적재
6Action Plan 확인 — action_plan_dashboard.html에서 자동 분할된 일자별 KPI 확인
KPI 총계 자동배분 안내
기획전 등록 시 총 RN/REV/ADR만 입력하면 시스템이 다음 규칙으로 일자·사업장별로 자동 분할합니다.
  • 주말 가중치 — 금/토/일 1.4배, 평일 1.0배
  • 공휴일 가중치 — 어린이날·현충일·광복절 등 1.6배 (2026 공휴일 캘린더 기준)
  • 사업장 가중치 — 패키지 코드 매핑 시 사업장별 평균 ADR 비율 적용
  • ADR 자동계산 — REV/RN 입력 시 ADR 자동 도출, 셋 중 둘만 입력해도 나머지 자동 산출
  • 중복 방지 — 동일 기획전명+패키지코드 조합은 Key시트에서 차단

주의사항

  • 패키지 코드는 Key시트의 사전 정의 코드만 허용 — 신규 코드는 GS팀 승인 후 등록
  • 총계 입력 후 자동배분 결과는 Action Plan 페이지에서 수정 가능 (개별 일자 오버라이드)
  • 제출 후 12시간 내 수정은 동일 기획전 ID로 덮어쓰기, 이후는 신규 버전으로 적재

개요

FCST(Forecast) 키인은 각 사업장 담당자가 향후 매출 예측 데이터를 입력하는 프로세스입니다. 정확한 예측은 전사 경영계획 수립의 핵심 기초 데이터가 됩니다.

키인 절차

1FCST 키인 페이지 접속 — 상단 "FCST 키인" 카드 또는 fcst-admin.html로 접속
2사업장 선택 — 담당 사업장을 드롭다운에서 선택
3기간 설정 — 예측 대상 월을 선택 (당월+1 ~ +3개월)
4데이터 입력 — 객실 매출, F&B, 부대시설 등 항목별 예측값 입력
5저장 및 확인 — 입력 데이터 검증 후 저장

주의사항

  • 키인 마감일: 매월 25일까지 완료
  • 수정이 필요한 경우, 마감일 전까지 자유롭게 재입력 가능
  • 전년 동기 대비 ±30% 이상 차이 시 사유 코멘트 필수
  • 키인 현황은 fcst-status.html에서 실시간 확인 가능

개요

품의서는 가격 조정, 특별 할인, 프로모션 등 영업 관련 의사결정을 위한 결재 문서입니다. 정확한 양식 작성이 신속한 결재의 핵심입니다.

작성 절차

1품의 작성 페이지 접속 — 상단 "품의 작성" 카드 또는 pumui.html로 접속
2품의 유형 선택 — 가격조정 / 프로모션 / 특별요청 등 유형 선택
3기본 정보 입력 — 대상 사업장, 적용 기간, 대상 상품 입력
4상세 내용 작성 — 조정 내역, 사유, 예상 효과 기술
5첨부 및 생성 — 관련 자료 첨부 후 품의서 PDF 생성

작성 팁

  • 사유란에 경쟁사 동향, 시장 상황 등 객관적 근거를 포함하면 결재 속도 향상
  • 예상 매출 효과를 수치화하여 기재 권장
  • 긴급 건의 경우 제목에 [긴급] 표기
  • 이전 승인 사례 참고 시 품의번호를 함께 기재
