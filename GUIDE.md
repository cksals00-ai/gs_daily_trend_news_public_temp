## ✏️ 담당자가 매일 수정하는 것 (단 하나의 파일)

👉 https://github.com/cksals00-ai/gs_daily_trend_news_public_temp/edit/main/data/daily_notes.json

### 수정할 필드: `executive_kpi`의 **3개 값**

```json
"executive_kpi": {
  "kpi_1": {
    "label": "🎯 비발디 권역 달성률",
    "value": "84.1",          ← 👈 오늘 Power BI 값
    "unit": "%",
    "delta": "▼ 소노캄 부진"  ← 👈 한 줄 메모 (선택)
  },
  "kpi_2": {
    "label": "⭐ 중부권 Pacing",
    "value": "108.4",         ← 👈 오늘 Power BI 값
    "unit": "%",
    "delta": "▲ 8.4% vs 목표"  ← 👈 한 줄 메모 (선택)
  },
  "kpi_3": {
    "label": "🏨 주의 사업장",
    "value": "5",             ← 👈 주의 사업장 수
    "unit": "/48",
    "delta": "▼ 소노캄비발디·삼척"  ← 👈 한 줄 메모 (선택)
  }
}
```

## 🚀 작업 순서

1. **GitHub에서 파일 열기** (위 링크 클릭)
2. **연필 아이콘** 클릭해서 편집 모드
3. `value` 3개 숫자 오늘 값으로 수정
4. (선택) `delta` 한 줄 메모 업데이트
5. 하단 **Commit message**: `daily: update YYYY-MM-DD` (날짜 바꾸기만)
6. **Commit changes** 버튼 클릭
7. 끝 ✨

자동으로:
- 🔥 오늘의 한 줄이 값에 따라 재작성됨
- 🎯 권역 액션이 다시 생성됨
- 📰 최신 뉴스가 상단에 반영됨
- 🚀 GitHub Pages 자동 재배포 (2~3분)

## 📊 KPI 값 해석 기준 (자동화 로직)

### KPI 1 & 2 (달성률/Pacing)
| 값 | 상태 | 자동 메시지 톤 |
|----|------|----------|
| 100% 이상 | 🟢 good | "목표 초과 · 호조" |
| 85~99% | 🟡 warn | "근접 · 주의 관찰" |
| 85% 미만 | 🔴 bad | "부진 · 긴급 대응" |

### KPI 3 (주의 사업장 수)
| 값 | 상태 | 자동 메시지 톤 |
|----|------|----------|
| 3곳 이하 | 🟢 good | "양호" |
| 4~7곳 | 🟡 warn | "관찰 중" |
| 8곳 이상 | 🔴 bad | "증가 · 긴급 대응" |
