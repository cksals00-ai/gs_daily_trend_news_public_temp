## 2. 차트 유형별 라이브러리 매핑

기존 트렌드 리포트가 이미 **Chart.js 4.4.0**을 사용 중이므로 동일 라이브러리 기반 통일이 최적.

| PBIX 차트 유형 | 사용 횟수 | 추천 HTML/JS 구현 | 라이브러리 |
|----------------|-----------|-------------------|------------|
| **카드 (card)** | 19회 | 기존 `signal-card` / `trend-now-value` 패턴 | 순수 HTML/CSS |
| **슬라이서 (slicer)** | 31회 | `<select>`, 커스텀 드롭다운, 날짜 피커 | 순수 JS + Flatpickr(날짜) |
| **피벗 테이블 (pivotTable)** | 8회 | AG Grid Free 또는 커스텀 `<table>` + JS 소계 | AG Grid Community |
| **KPI 표시기** | 5회 | `strategy-progress-pct` + `strategy-bar` 패턴 활용 | 순수 HTML/CSS |
| **콤보 차트 (line+bar)** | 3회 | Chart.js mixed chart (`type: 'bar'` + line dataset) | **Chart.js** (이미 로드됨) |
| **파이 차트** | 6회 | Chart.js `type: 'pie'` | **Chart.js** |
| **도넛 차트** | 3회 | Chart.js `type: 'doughnut'` | **Chart.js** |
| **영역 차트** | 2회 | Chart.js line chart with `fill: true` | **Chart.js** |
| **게이지** | 1회 | Chart.js doughnut (180° rotation) 또는 CSS arc | **Chart.js** 또는 인라인 SVG |
| **스택 콤보** | 1회 | Chart.js stacked bar + line overlay | **Chart.js** |
| **테이블 (tableEx)** | 5회 | 기존 다크 테이블 CSS 활용 | 순수 HTML/CSS |

### 라이브러리 스택 추천

| 구분 | 1순위 (경량) | 2순위 (고기능) |
|------|-------------|---------------|
| **차트 전체** | Chart.js 4.x (이미 사용 중) | ECharts 5.x (피벗 차트 연동 강점) |
| **피벗 테이블** | 커스텀 `<table>` + JS 소계 로직 | AG Grid Community (무료, 정렬/필터 내장) |
| **날짜 필터** | Flatpickr (6KB) | — |
| **게이지** | Chart.js doughnut hack | SVG 인라인 (디자인 자유도 높음) |

## 3. 데이터 소스 전략

### 현재 PBIX 연결 구조
```
Power BI Service (Cloud)
  └─ Dataset ID: 8ee000d9-5efb-403f-83ad-9a8e3d3b80eb
     └─ DataModel (492MB, Vertipaq/XPRESS9 압축)
        └─ 원본 소스: 암호화되어 직접 확인 불가
```

### 방안 A: Power BI REST API 활용 (권장 — 초기)

| 항목 | 내용 |
|------|------|
| **방법** | Power BI REST API → `executeQueries` 엔드포인트로 DAX 쿼리 실행 |
| **장점** | 기존 데이터셋 그대로 활용, 추가 ETL 불필요 |
| **단점** | API 호출 제한 (시간당 제한), 실시간성 제약, 인증 토큰 관리 필요 |
| **인증** | Azure AD App Registration → Service Principal 또는 Master User 토큰 |
| **적합 시나리오** | 일 1~2회 갱신, 사용자 수 소규모 |

```
[HTML Dashboard] → [중간 서버 (Node/Python)]
                     ↓
              Power BI REST API
              POST /datasets/{id}/executeQueries
              Body: { "queries": [{ "query": "EVALUATE SUMMARIZE(...)" }] }
```

### 방안 B: 별도 데이터 파이프라인 (장기 — 확장성)

| 항목 | 내용 |
|------|------|
| **방법** | 원본 DB/Excel → ETL → JSON API 서버 → HTML Dashboard |
| **장점** | Power BI 의존성 제거, 완전한 제어, 실시간 가능 |
| **단점** | 원본 소스 파악 필요, 초기 구축 비용 |
| **적합 시나리오** | 다수 사용자, 실시간 갱신, Power BI 라이선스 절감 목적 |

### 방안 C: 하이브리드 (현실적 추천)

1단계에서 **Power BI API + 결과 캐싱(JSON 파일)**으로 시작하고, 이후 원본 소스를 파악하여 직접 연결로 전환.

```
[스케줄러 (매일 06:00)]
  → Power BI API로 DAX 쿼리 실행
  → 결과를 JSON 파일로 저장 (/data/otb_2026.json)
  → GitHub Pages 또는 정적 호스팅에 배포

[HTML Dashboard]
  → fetch('/data/otb_2026.json')
  → Chart.js / 테이블 렌더링
```

## 4. 기존 톤앤매너 호환성 분석

### 기존 트렌드 리포트 디자인 시스템 요약

| 요소 | 기존 값 | OTB 대시보드 활용 |
|------|---------|------------------|
| **배경** | `--bg: #1a1d23` (차콜 네이비) | 그대로 적용 |
| **카드** | `--bg-card: #252932`, border-radius 4~6px | 카드/KPI에 동일 적용 |
| **골드 액센트** | `--gold: #c9a063`, `--gold-bright: #e6b970` | 달성률 게이지, 헤더 강조에 활용 |
| **Positive/Negative** | 녹색 `#4caf6f` / 빨강 `#e57373` | 증감률, YoY 비교에 바로 매핑 |
| **폰트** | Pretendard + Noto Sans KR + JetBrains Mono | 본문/숫자/라벨에 동일 3종 유지 |
| **차트 스타일** | height 52px 미니 라인, 다크 배경 | OTB용은 더 큰 차트 필요 → 높이 확대 |

### 재활용 가능한 기존 컴포넌트

| 기존 컴포넌트 | OTB 대시보드 용도 |
|---------------|------------------|
| `.signal-card` (3-column grid, dot + name + value) | → 사업장별 KPI 카드 (RNS, REV, ADR) |
| `.trend-card` (big number + delta + sparkline) | → 오늘 순증/예약/취소 RNS 카드 |
| `.trend-now-value` + `.trend-now-delta` | → 달성률 대형 숫자 표시 |
| `.strategy-card` (header + progress bar) | → RNS/REV 달성률 게이지 대체 |
| `.strategy-bar` + `.strategy-bar-fill` | → 예산 대비 실적 진행률 바 |
| `.region-card` (border-top color coding) | → 사업장별 컬러 코딩 카드 |
| Chart.js 미니 라인 차트 (canvas) | → 예약 추이 스파크라인 |
| `.masthead` (sticky header + live dot) | → OTB 대시보드 네비게이션 바 |

### 신규 구현 필요 컴포넌트

| 컴포넌트 | 설명 | 구현 방향 |
|----------|------|-----------|
| **피벗 테이블** | 행×열 교차 + 소계 | 다크 테마 커스텀 `<table>`, `--bg-card` 배경, 홀짝 행 `--bg-soft` |
| **이중 Y축 콤보 차트** | 막대(실적) + 라인(달성률) | Chart.js mixed chart, 골드 라인 + 반투명 바 |
| **게이지 (반원형)** | RNS 달성률 시각화 | SVG arc 또는 Chart.js doughnut, 골드 컬러 |
| **슬라이서 패널** | 사업장/월/세그 필터 | 다크 드롭다운, `--bg-soft` 배경, 골드 포커스 링 |
| **페이지 네비게이터** | 11개 페이지 탭/라우팅 | 탭 바 또는 사이드 네비, `--gold` 액티브 표시 |

### 호환성 판정: ★★★★★ (매우 높음)

기존 리포트가 이미 호텔/리조트 KPI 표현에 최적화된 다크 디자인 시스템을 갖추고 있어, CSS 변수와 컴포넌트의 **70% 이상을 그대로 재활용** 가능합니다. 신규 구현이 필요한 것은 피벗 테이블, 게이지, 슬라이서 UI 정도입니다.
